# Adaptive Math Step Analyser

Backend-сервис для пошагового анализа математических решений, диагностики типовых ошибок и адаптивного подбора следующей задачи.

## О проекте

**Adaptive Math Step Analyser** — это backend-модуль для образовательной системы, в которой ученик вводит решение задачи по шагам, а система:

* разбирает математический ввод;
* проверяет корректность преобразования;
* выявляет типичные ошибки;
* формирует текстовую обратную связь;
* обновляет профиль обучающегося;
* рекомендует следующую задачу с учётом истории ошибок.

Проект ориентирован на сценарий адаптивного обучения математике.

---

## Основные возможности

* REST API на **FastAPI**
* хранение данных в **PostgreSQL**
* миграции через **Alembic**
* анализ математических шагов через **SymPy**
* диагностика типовых ошибок:

  * `sign_error`
  * `arithmetic_error`
  * `invalid_transform`
  * `parse_error`
  * `math_parse_error`
  * `possible_domain_loss`
  * `division_by_zero`
* формирование обратной связи по шагу
* накопление статистики по профилю ученика
* рекомендации по следующей задаче
* тесты на API и бизнес-логику

## Архитектура

Проект разделён на несколько уровней:

* `app/api/routers` — HTTP API
* `app/services` — бизнес-логика, диагностика, рекомендации
* `app/math_core` — математический разбор и анализ переходов
* `app/models` — SQLAlchemy-модели
* `app/schemas` — Pydantic-схемы
* `alembic` — миграции базы данных
* `tests` — тесты

### Логика обработки шага

При отправке шага решения система выполняет следующий пайплайн:

1. нормализация и разбор текста шага;
2. определение формата:

   * одиночное выражение;
   * переход вида `до => после`;
3. математический анализ преобразования;
4. проверка ограничений;
5. диагностика типа ошибки;
6. генерация обратной связи;
7. сохранение результата анализа;
8. обновление профиля ученика;
9. подбор следующей задачи.

## Основные API-маршруты

### Health

* `GET /health`

Проверка работоспособности сервиса.

---

### Problems

* `POST /problems`
* `GET /problems/{problem_id}`

Создание и получение задач.

---

### Attempts

* `POST /attempts`
* `GET /attempts/{attempt_id}`
* `POST /attempts/{attempt_id}/steps`

Создание попытки решения и отправка шагов.

---

### Profiles

* `GET /profiles/{profile_key}`

Получение профиля ученика, статистики и рекомендаций.

---

### Practice

* `GET /practice/next-problem/{profile_key}`
* `POST /practice/generate-problem`

Подбор следующей задачи на основе профиля или фильтров.

---

### Admin

* `POST /admin/profiles/{profile_key}/reset`
* `POST /admin/seed-default-problems`

Сброс профиля и сидирование тестовых задач.

## Пример логики анализа

Для шага:

```text
2x + 6 = 10 => 2x = 10 - 6
```

система может:

* распознать формат перехода;
* проверить равносильность;
* выставить оценки `math_score`, `logic_score`, `condition_score`, `goal_score`;
* определить `diagnosis_code`;
* вернуть `feedback`;
* сохранить результат;
* обновить статистику профиля.

Для шага с ошибкой знака:

```text
2x + 6 = 10 => 2x = 10 + 6
```

ожидается диагностика типа `sign_error` и объясняющая обратная связь.
